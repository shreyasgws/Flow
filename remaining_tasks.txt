P12 — ANIMATION SYSTEM (TIER-ADAPTIVE PERFORMANCE)
====================================================
Reference: flow_animation_system.txt

Complete the tier-adaptive animation system that preserves emotional
quality across all devices by translating (not removing) effects.

  — DEVICE TIER DETECTION —
  - GPU benchmark: quick canvas test (<8ms = capable)
  - Reads navigator.deviceMemory, hardwareConcurrency, connection type
  - Respects prefers-reduced-motion and Save-Data mode
  - Caches result in localStorage for 7 days
  - Three tiers: FULL (flagship), STANDARD (mid-range), MINIMAL (low-end)
  - Write dataset-tier to <html> so CSS reads it

  — CSS TOKEN SYSTEM PER TIER —
  - Ambient tokens: --ambient-layer-count, --ambient-*-op, --ambient-duration-*
  - Blur tokens: --blur-composer, --blur-sheet, --blur-nav
  - Spring physics tokens: --spring-stiffness, --spring-damping
  - Shadow tokens per tier
  - Stagger tokens: --stagger-delay, --stagger-task
  - Focus breathe tokens: --focus-breathe-duration
  - Timeline tokens: --timeline-pulse-duration
  - Dandelion tokens: --dandelion-animate, --dandelion-burst
  - [data-tier="standard"] and [data-tier="minimal"] overrides

  — TIER-ADAPTIVE AMBIENT BACKGROUND —
  - FULL: 5 layers (base + silhouette + haze + bloom + grain), animated
  - STANDARD: 2 layers (base + bloom), static (no animation)
  - MINIMAL: single background color, no layers

  — ANIMATION LAYER ENFORCEMENT —
  - Layer 1 (Interaction): NEVER reduced on any tier
    (swipe, checkmark fill, card expand, sheet open, etc.)
  - Layer 2 (Response): full on all tiers, stagger disabled on Minimal
  - Layer 3 (Ambient): tier-scaled (5/2/1 layers)
  - Layer 4 (Microstate): all tiers, shorter durations on Minimal

  — TIER-ADAPTIVE DANDELION —
  - FULL: canvas, full breathing + tap burst
  - STANDARD: canvas, breathing only (no burst)
  - MINIMAL: static SVG (identical visual, zero animation cost)

  — MOTION CONFIG PER TIER —
  - FULL: spring (stiffness 120, damping 14)
  - STANDARD: spring (stiffness 100, damping 16)
  - MINIMAL: linear ease 150ms

  — BLUR REPLACEMENT —
  - FULL: backdrop-filter blur active
  - STANDARD/MINIMAL: solid backgrounds with border replacing blur
  - CSS overrides via [data-tier="standard"] .composer, .sheet, .nav

  — TRANSITION PAUSING —
  - Pause ambient animations during page navigation
  - Pause ambient + microstate during drag-to-reorder
  - Pause ambient + microstate (except focusBreathe) during focus mode
  - Resume after transition completes

  — PERFORMANCE MONITORING —
  - FPS detection on STANDARD tier only
  - If <25fps for 5 consecutive seconds, mark for downgrade next session
  - Never change tiers mid-session (avoids jarring shifts)

  — BATTERY SAVER —
  - If battery <15% or charging=false + battery <20%, treat as Tier 3
  - Detection runs only on session start

  — FOCUS MODE PER TIER —
  - FULL: breathing animation, bloom gradient
  - STANDARD: static gradient (no breathing)
  - MINIMAL: single deep background color

  — ANIMATION CONCURRENCY BUDGET —
  - Layer 1: max 2 simultaneous
  - Layer 2: max 4 simultaneous
  - Layer 3: max 4 CSS keyframes
  - Layer 4: max 4 simultaneous
  - Total: max 14 animated elements at once

  — VIRTUAL TASK LIST (moved from P10) —
  - Only render tasks in/near viewport when count > 15
  - Estimate height, overscan 3 items
  - For typical days (4-8 tasks), render all normally

P13 — VISUAL FIDELITY & BIBLE DESIGN MATCH
===========================================
Make every UI element, animation, and visual detail match the Bible spec.
This is the polish pass that transforms a functional app into a beautiful
one that feels like the product described in the design bible.

  — DRIFT BUTTON (DANDELION ANIMATION) —
  - Drift page button: animated canvas dandelion (not static SVG)
  - Breathing animation when idle (slow scale pulse + rotation drift)
  - Tap burst: seeds scatter outward with fade + fall physics
  - Tier-adaptive: canvas on Full/Standard, static SVG on Minimal
  - Fallback: if canvas fails, degrade to static SVG gracefully
  - Color matches Bible spec (warm glow, #d4a373 / #faedcd palette)

  — GLASS MORPHISM (BACKDROP-FILTER BLUR) —
  - Bottom sheets: blur bg, translucent border, subtle shadow
  - Navigation bar: frosted glass effect matching Bible spec
  - Composer: blurred background when open
  - Cards: subtle glass sheen on hover/tap
  - Standard/Minimal tiers: fallback to solid bg + border per P12

  — SPRING PHYSICS ON ALL INTERACTIONS —
  - Card expand/collapse: spring curve (stiffness 120, damping 14)
  - Sheet open/close: spring with bounce
  - Task checkmark fill: elastic spring on complete
  - Swipe-to-delete: spring snap to thresholds
  - Button press: subtle scale spring (0.98 → 1.0)
  - All easing replaces linear/bezier transitions

  — STAGGER ANIMATIONS —
  - Weekly cards: appear with 80ms stagger delay per card
  - Task list: items fade-slide in with 40ms stagger
  - Reflection prompts: staggered entrance
  - Section headers: fade in before content
  - Page load: content fades in after shell renders
  - Respects prefers-reduced-motion (immediate reveal)

  — PAGE TRANSITIONS —
  - Crossfade between route changes (300ms)
  - Ambient background pauses during transition
  - Content slides up slightly on enter
  - Back navigation reverses the animation
  - Focus mode enter: smooth dim + zoom transition
  - Review page: slide-in from bottom

  — BIBLE DESIGN TOKEN MATCH —
  - Audit every color against Bible palette
  - Typography: ensure font sizes, weights, line-height match spec
  - Spacing: verify padding/margins against Bible spacing scale
  - Button styles: border radius, height, font, shadow per spec
  - Iconography: replace any placeholder icons with Bible-spec ones
  - Shadows: apply design token shadows to cards, sheets, nav
  - Borders: use Bible-spec border colors and widths
  - Empty states: illustrated placeholders matching Bible design

  — AMBIENT BACKGROUND FIDELITY —
  - Ensure 5-layer ambient (base + silhouette + haze + bloom + grain)
  - Silhouette: abstract gentle shapes, parallax on tilt
  - Haze: layered opacity, slow horizontal drift
  - Bloom: radial glow behind content, pulsing slowly
  - Grain: subtle noise overlay (CSS or SVG filter)
  - All animated with varying durations (12s–30s cycles)
  - No layer on Minimal tier; solid bg only

  — MICRO-INTERACTIONS POLISH —
  - Settings toggle: smooth thumb slide with spring
  - Slider: animated fill track
  - Segmented control: animated active tab underline
  - Pull-to-refresh: rubber-band resistance + spring snap
  - Drag handle: subtle glow on hover
  - Long press: haptic-like visual ripple
  - Loading skeleton: shimmer animation per Bible spec

  — KEYBOARD & REDUCED MOTION —
  - Reduced motion: all animations disabled, immediate transitions
  - Keyboard focus: visible ring matches Bible spec (not browser default)
  - Tab order verified across all interactive elements
  - Focus trap in modals/sheets with animation
