PART ONE — THE PHILOSOPHY
---
The Core Principle
Animations in Flow are not decoration. They are communication.
Every animation either communicates a state change, provides spatial orientation,
reinforces emotional identity, or gives tactile feedback.
If an animation does none of these four things, it should not exist.
The goal is not "fewer animations on low-end devices."
The goal is "the same emotional experience rendered in cheaper materials."
A warm amber glow at dawn can be:
5 animated GPU-composited layers on a flagship device
A single warm background color transition on a low-end device
Both feel like morning. Both feel like Flow.
One costs 2% GPU. The other costs 0.1%.
The user feels the same thing.
This distinction — same feeling, different cost — is the entire philosophy
of the performance system. Nothing is removed. Everything is translated.
---
What "Beautiful" Actually Means on Low-End Devices
A low-end phone user is not expecting desktop-class effects.
They are expecting the app to feel:
Instant
Smooth
Intentional
Calm
A low-end phone running a stripped-down but perfectly smooth Flow
feels more beautiful than a flagship phone running Flow at 25fps
because of over-ambitious blur and particle effects.
Smoothness IS beauty on mobile.
Jank destroys trust faster than any missing visual effect.
The rule: a low-end device should never feel like it is running
a degraded version of Flow. It should feel like it is running
the right version of Flow for its capabilities.

PART TWO — DEVICE TIER DETECTION
---
The Three Tiers
```
Tier 1 — FULL
  Flagship phones (iPhone 14+, Pixel 7+, Samsung S22+)
  Desktop and laptop browsers
  All 5 ambient layers animated
  Backdrop-filter blur active
  Spring physics on all gestures
  Canvas dandelion with full breathing + burst
  Particle effects
  Grain texture
  Full stagger animations

Tier 2 — STANDARD
  Mid-range phones (Snapdragon 700 series, iPhone XR–13)
  Most Android phones 2019–2022
  2 ambient layers (color + bloom), static
  No backdrop-filter blur (solid backgrounds)
  Spring physics on gestures (reduced stiffness)
  Canvas dandelion breathing only (no burst particle trail)
  No particles
  No grain texture
  Stagger animations preserved (they are cheap)

Tier 3 — MINIMAL
  Low-end phones (Snapdragon 600 and below, older iPhones pre-XR)
  Any device with prefers-reduced-motion
  Any device with Save Data mode
  Single background color per time period (no layers)
  No backdrop-filter ever
  No spring physics (linear ease 200ms)
  Static dandelion SVG (no canvas, no animation)
  No particles, no grain, no bloom
  No stagger (all elements appear simultaneously)
  Only interaction animations remain (essential only)
```

Detection Logic
Create this file first before any component is written.
It runs once at startup. Result is locked for the session.
```javascript
// /src/lib/deviceTier.js

export const TIERS = {
  FULL: 'full',
  STANDARD: 'standard',
  MINIMAL: 'minimal'
}

function runGPUBenchmark() {
  // Quick canvas benchmark — draws 1000 gradient operations
  // Returns true if device can handle it in under 8ms
  try {
    const canvas = document.createElement('canvas')
    canvas.width = 200
    canvas.height = 200
    const ctx = canvas.getContext('2d')
    const start = performance.now()
    for (let i = 0; i < 100; i++) {
      const grad = ctx.createRadialGradient(100, 100, 0, 100, 100, 100)
      grad.addColorStop(0, 'rgba(200,131,42,0.1)')
      grad.addColorStop(1, 'rgba(0,0,0,0)')
      ctx.fillStyle = grad
      ctx.fillRect(0, 0, 200, 200)
    }
    const elapsed = performance.now() - start
    return elapsed < 8
  } catch {
    return false
  }
}

function detectTier() {
  // Immediate minimal signals — no benchmark needed
  const prefersReducedMotion = window.matchMedia(
    '(prefers-reduced-motion: reduce)'
  ).matches

  const saveData = navigator.connection?.saveData === true

  const slowConnection = ['slow-2g', '2g'].includes(
    navigator.connection?.effectiveType
  )

  if (prefersReducedMotion || saveData || slowConnection) {
    return TIERS.MINIMAL
  }

  // Hardware signals
  const memory = navigator.deviceMemory || 4      // GB, defaults 4 if not available
  const cores = navigator.hardwareConcurrency || 4

  if (memory <= 1 || cores <= 2) {
    return TIERS.MINIMAL
  }

  if (memory <= 3 || cores <= 4) {
    // Run quick benchmark to decide between minimal and standard
    const gpuCapable = runGPUBenchmark()
    return gpuCapable ? TIERS.STANDARD : TIERS.MINIMAL
  }

  // Higher end — benchmark to decide between standard and full
  const gpuCapable = runGPUBenchmark()
  return gpuCapable ? TIERS.FULL : TIERS.STANDARD
}

// Run detection once, lock it
export const DEVICE_TIER = detectTier()

// Write to document so CSS can read it
document.documentElement.dataset.tier = DEVICE_TIER

// Write to localStorage so subsequent sessions skip benchmark
// (benchmark result cached for 7 days)
try {
  const cached = localStorage.getItem('flow_device_tier')
  const cachedTime = localStorage.getItem('flow_device_tier_time')
  const sevenDays = 7 * 24 * 60 * 60 * 1000

  if (cached && cachedTime && Date.now() - Number(cachedTime) < sevenDays) {
    // Use cached result — but always respect prefers-reduced-motion in real time
    const tier = (prefersReducedMotion || saveData) ? TIERS.MINIMAL : cached
    document.documentElement.dataset.tier = tier
  } else {
    localStorage.setItem('flow_device_tier', DEVICE_TIER)
    localStorage.setItem('flow_device_tier_time', String(Date.now()))
  }
} catch {
  // localStorage unavailable — use detected tier
}

// Export a hook for React components
export function useDeviceTier() {
  return DEVICE_TIER
}

// Export boolean helpers
export const isFull = DEVICE_TIER === TIERS.FULL
export const isStandard = DEVICE_TIER === TIERS.STANDARD
export const isMinimal = DEVICE_TIER === TIERS.MINIMAL
export const hasSpring = !isMinimal
export const hasBlur = isFull
export const hasAmbient = !isMinimal
export const hasCanvas = !isMinimal
export const hasParticles = isFull
export const hasGrain = isFull
export const hasStagger = !isMinimal
```

PART THREE — CSS TOKEN SYSTEM
---
The Token Architecture
Every visual property in Flow reads from a CSS custom property.
Components never hardcode animation values, durations, blur amounts,
or shadow values. They always use tokens.
When the device tier changes a token value, every component
that reads that token adapts automatically.
No component needs to know what tier it is on.

PART FOUR — THE FOUR ANIMATION LAYERS IN DETAIL
---
Layer 1 — Interaction Animations
These are triggered directly by user action.
The user is watching these happen.
They must be perfect on every device including Tier 3.
They are the non-negotiable beauty of Flow.
What they are:
```
Task completion (checkmark fill + title strike)    200ms
Card expand / collapse                             200ms ease-out
Swipe gesture follow                               Real-time, no lag
Drag lift                                          150ms
Drag drop                                          300ms spring
Composer expand                                    300ms spring
Bottom sheet open                                  350ms spring
Bottom sheet close                                 280ms ease-in
Drift sheet open                                   300ms spring
Drift sheet close                                  250ms ease-in
Page transitions                                   280ms ease-out
Focus mode entry                                   400ms
Focus mode exit                                    350ms ease-out
```
Implementation rules:
CSS `transform` and `opacity` only. Never width, height, top, left, margin
`will-change: transform` on every element that animates
`translateZ(0)` to force GPU layer promotion
On Tier 3: same animations, shorter durations (see token table), linear easing
These animations NEVER get disabled. A Tier 3 device still gets the
checkmark fill, the card expand, the sheet spring.
They are just slightly shorter and linear instead of spring-curved.
This is the difference between "snappy" and "broken."
Task Completion Animation (works on all tiers):
```css
/* The checkmark fills with section color */
.task-checkbox {
  transition: background-color var(--dur-fast) var(--ease-default),
              border-color var(--dur-fast) var(--ease-default);
  will-change: transform;
}

/* Title strikes through */
.task-title-done {
  transition: opacity var(--dur-fast) var(--ease-out),
              text-decoration var(--dur-fast);
}

/* Card settles */
.task-card-done {
  transition: opacity var(--dur-default) var(--ease-out);
}
```
On Tier 3: `--dur-fast` is 100ms and `--ease-default` is linear.
The animation still happens. It is just shorter and linear.
Still feels intentional. Not broken.

Layer 2 — Response Animations
Triggered by data changes or system events.
Not continuous. Fire once, complete, go idle.
Cost is near-zero when not firing.
What they are:
```
Task creation fade-in              300ms, then stops
Weekly card stagger on page load   40ms × 7, then stops completely
Day Pulse fade-in on load          200ms, then stops
Carry forward banner slide-in      250ms, then stops
Soft Intelligence suggestion fade  300ms, then stops
Session transition prompt fade     300ms, then stops
Timeline line state change         400ms, then stops
Drift item creation                300ms, then stops
Badge count bump on new drift item 150ms scale, then stops
Sync indicator state changes       200ms, then stops
```
Key insight:
These animations are free most of the time because they are not running.
A task creation animation costs GPU for 300ms out of every hour the user
uses the app. The total amortized cost is negligible.
On Tier 3, stagger is removed (all at 0ms delay) but the animations
themselves still run. Tasks still fade in. Sheets still open.
The only difference is that 7 weekly cards appear at the same time
instead of with 40ms delays.
Stagger implementation:
```css
/* Full and Standard — staggered */
.weekly-card:nth-child(1) { animation-delay: calc(0 * var(--stagger-weekly)); }
.weekly-card:nth-child(2) { animation-delay: calc(1 * var(--stagger-weekly)); }
.weekly-card:nth-child(3) { animation-delay: calc(2 * var(--stagger-weekly)); }
.weekly-card:nth-child(4) { animation-delay: calc(3 * var(--stagger-weekly)); }
.weekly-card:nth-child(5) { animation-delay: calc(4 * var(--stagger-weekly)); }
.weekly-card:nth-child(6) { animation-delay: calc(5 * var(--stagger-weekly)); }
.weekly-card:nth-child(7) { animation-delay: calc(6 * var(--stagger-weekly)); }

/* On Tier 3: --stagger-weekly = 0ms so all delays are 0 */
/* All 7 cards appear simultaneously */
```

Layer 3 — Ambient Animations
Always running. The atmosphere of Flow.
This is where the visual beauty lives.
This is also where the performance risk lives.
The fix: tier-adaptive rendering with CSS-only GPU-composited motion.
Full Tier Implementation (5 layers):
```jsx
// /src/components/layout/AmbientBackground.jsx

import { useMemo, useEffect, useState } from 'react'
import { isFull, isStandard, isMinimal } from '../../lib/deviceTier'

function getTimeOfDay() {
  const hour = new Date().getHours()
  if (hour >= 6 && hour < 12) return 'morning'
  if (hour >= 12 && hour < 17) return 'afternoon'
  if (hour >= 17 && hour < 21) return 'evening'
  return 'night'
}

const TIME_CONFIG = {
  morning: {
    base: '#0F0E0C',
    bloom: '#C8832A',
    bloomPosition: '80% 20%',
    bloomOpacity: 0.08,
  },
  afternoon: {
    base: '#0D0D11',
    bloom: '#4A7FA5',
    bloomPosition: '50% 50%',
    bloomOpacity: 0.06,
  },
  evening: {
    base: '#0B0D12',
    bloom: '#3A5A7A',
    bloomPosition: '20% 80%',
    bloomOpacity: 0.08,
  },
  night: {
    base: '#09090F',
    bloom: '#4A3070',
    bloomPosition: '50% 60%',
    bloomOpacity: 0.07,
  },
}

export default function AmbientBackground() {
  const [timeOfDay, setTimeOfDay] = useState(getTimeOfDay)

  // Update time of day every 60 seconds
  useEffect(() => {
    const interval = setInterval(() => {
      setTimeOfDay(getTimeOfDay())
    }, 60000)
    return () => clearInterval(interval)
  }, [])

  const config = TIME_CONFIG[timeOfDay]

  // Tier 3: just a single background color
  // Simple, clean, still feels like morning/evening
  if (isMinimal) {
    return (
      <div
        className="ambient-root"
        style={{
          position: 'fixed',
          inset: 0,
          zIndex: 0,
          pointerEvents: 'none',
          backgroundColor: config.base,
          transition: `background-color var(--dur-ambient) linear`,
        }}
      />
    )
  }

  // Tier 2: base color + bloom gradient only, static
  if (isStandard) {
    return (
      <div
        className="ambient-root"
        style={{
          position: 'fixed',
          inset: 0,
          zIndex: 0,
          pointerEvents: 'none',
          backgroundColor: config.base,
          transition: `background-color var(--dur-ambient) linear`,
        }}
      >
        {/* Layer 4: bloom — single radial gradient, no animation */}
        <div
          style={{
            position: 'absolute',
            inset: 0,
            background: `radial-gradient(
              ellipse 60% 60% at ${config.bloomPosition},
              ${config.bloom}${Math.round(config.bloomOpacity * 255).toString(16).padStart(2,'0')} 0%,
              transparent 70%
            )`,
            // No animation on standard — static bloom
            // Still creates depth and atmosphere
            // Just not breathing
          }}
        />
      </div>
    )
  }

  // Tier 1: all 5 layers, animated
  return (
    <div
      className="ambient-root"
      style={{
        position: 'fixed',
        inset: 0,
        zIndex: 0,
        pointerEvents: 'none',
        backgroundColor: config.base,
        transition: `background-color var(--dur-ambient) linear`,
      }}
    >
      {/* Layer 2: Silhouette blur shape */}
      <div
        className="ambient-silhouette"
        style={{
          position: 'absolute',
          inset: 0,
          background: `radial-gradient(
            ellipse 80% 40% at 50% 70%,
            ${config.bloom}14 0%,
            transparent 60%
          )`,
          filter: 'blur(40px)',
          opacity: 'var(--ambient-silhouette-op)',
          animation: 'ambientDrift1 30s ease-in-out infinite',
          animationPlayState: 'var(--ambient-play-state)',
          willChange: 'transform',
        }}
      />

      {/* Layer 3: Haze gradient */}
      <div
        className="ambient-haze"
        style={{
          position: 'absolute',
          inset: 0,
          background: `radial-gradient(
            ellipse 100% 80% at ${config.bloomPosition},
            ${config.bloom}0D 0%,
            transparent 60%
          )`,
          filter: 'blur(60px)',
          opacity: 'var(--ambient-haze-op)',
          animation: 'ambientDrift2 40s ease-in-out infinite',
          animationPlayState: 'var(--ambient-play-state)',
          willChange: 'transform',
        }}
      />

      {/* Layer 4: Bloom */}
      <div
        className="ambient-bloom"
        style={{
          position: 'absolute',
          inset: 0,
          background: `radial-gradient(
            circle 200px at ${config.bloomPosition},
            ${config.bloom}08 0%,
            transparent 70%
          )`,
          opacity: 'var(--ambient-bloom-op)',
          animation: 'ambientDrift3 25s ease-in-out infinite',
          animationPlayState: 'var(--ambient-play-state)',
          willChange: 'transform',
        }}
      />

      {/* Layer 5: Grain texture — static SVG tile, no animation */}
      <div
        className="ambient-grain"
        style={{
          position: 'absolute',
          inset: 0,
          backgroundImage: `url("data:image/svg+xml,%3Csvg viewBox='0 0 200 200' xmlns='http://www.w3.org/2000/svg'%3E%3Cfilter id='noise'%3E%3CfeTurbulence type='fractalNoise' baseFrequency='0.65' numOctaves='3' stitchTiles='stitch'/%3E%3C/filter%3E%3Crect width='100%25' height='100%25' filter='url(%23noise)' opacity='1'/%3E%3C/svg%3E")`,
          backgroundSize: '200px 200px',
          opacity: 'var(--ambient-grain-op)',
          // No animation on grain — static texture
          // This is intentional: animated grain = noise = GPU expensive
        }}
      />

      {/* Particles: only on Tier 1 */}
      <AmbientParticles />
    </div>
  )
}
```

```jsx
            willChange: 'transform',
          }}
```

```jsx
// Canvas version — breathing animation (Tier 2 and Tier 1)
// Burst animation only on Tier 1 (--dandelion-burst = 1)
function DandelionCanvas({ triggerBurst }) {
  const canvasRef = useRef(null)
  const stateRef = useRef({
    t: 0,
    bursting: false,
    burstT: 0,
    rafId: null,
  })

  const stems = [
    { angle: -Math.PI/2,    baseLen: 11, baseOp: 0.92, phase: 0 },
    { angle: -Math.PI/4,    baseLen: 10, baseOp: 0.76, phase: Math.PI*2/8*1 },
    { angle: 0,             baseLen: 10, baseOp: 0.60, phase: Math.PI*2/8*2 },
    { angle: Math.PI/4,     baseLen: 9,  baseOp: 0.44, phase: Math.PI*2/8*3 },
    { angle: Math.PI/2,     baseLen: 8,  baseOp: 0.30, phase: Math.PI*2/8*4 },
    { angle: 3*Math.PI/4,   baseLen: 9,  baseOp: 0.26, phase: Math.PI*2/8*5 },
    { angle: Math.PI,       baseLen: 10, baseOp: 0.50, phase: Math.PI*2/8*6 },
    { angle: -3*Math.PI/4,  baseLen: 10, baseOp: 0.66, phase: Math.PI*2/8*7 },
  ]

  const CX = 20, CY = 20

  const easeOutCubic = x => 1 - Math.pow(1 - x, 3)
  const springFn = x => {
    const s = 1.5
    return Math.pow(2, -10*x) * Math.sin((x - s/4) * (2*Math.PI) / s) + 1
  }

  useEffect(() => {
    const canvas = canvasRef.current
    if (!canvas) return
    const ctx = canvas.getContext('2d')
    const dpr = Math.min(window.devicePixelRatio || 1, 2) // cap at 2x
    canvas.width = 40 * dpr
    canvas.height = 40 * dpr
    canvas.style.width = '40px'
    canvas.style.height = '40px'
    ctx.scale(dpr, dpr)

    function draw() {
      const state = stateRef.current
      ctx.clearRect(0, 0, 40, 40)

      state.t += 0.022

      // Burst state machine
      let extraLen = 0
      let opMul = 1

      if (state.bursting) {
        state.burstT += 0.016
        const BURST_OUT = 0.38
        const BURST_HOLD = 0.12
        const BURST_IN = 0.55
        const total = BURST_OUT + BURST_HOLD + BURST_IN

        if (state.burstT >= total) {
          state.bursting = false
          state.burstT = 0
        } else if (state.burstT < BURST_OUT) {
          const p = state.burstT / BURST_OUT
          extraLen = easeOutCubic(p) * 9
          opMul = 1 - easeOutCubic(p) * 0.3
        } else if (state.burstT < BURST_OUT + BURST_HOLD) {
          extraLen = 9
          opMul = 0.7
        } else {
          const p = (state.burstT - BURST_OUT - BURST_HOLD) / BURST_IN
          const springVal = springFn(p)
          extraLen = Math.max(0, (1 - springVal) * 9)
          opMul = 0.7 + springVal * 0.3
        }
      }

      // Draw stems
      stems.forEach((stem, i) => {
        const sway = Math.sin(state.t + stem.phase) * 0.07
        const breathe = Math.sin(state.t * 0.7 + stem.phase) * 1.3
        const staggeredExtra = extraLen * (1 + i * 0.03)

        const angle = stem.angle + sway
        const len = stem.baseLen + breathe + staggeredExtra

        const ex = CX + Math.cos(angle) * len
        const ey = CY + Math.sin(angle) * len

        // Stem line
        ctx.beginPath()
        ctx.moveTo(CX, CY)
        ctx.lineTo(ex, ey)
        ctx.strokeStyle = `rgba(240,239,232,${(stem.baseOp * opMul * 0.75).toFixed(3)})`
        ctx.lineWidth = 1
        ctx.lineCap = 'round'
        ctx.stroke()

        // Tip circle
        const tipR = 1.2 + Math.sin(state.t * 0.9 + stem.phase) * 0.35
        ctx.beginPath()
        ctx.arc(ex, ey, tipR, 0, Math.PI * 2)
        ctx.fillStyle = `rgba(240,239,232,${(stem.baseOp * opMul).toFixed(3)})`
        ctx.fill()
      })

      // Center core
      const coreR = 2.1 + Math.sin(state.t * 1.1) * 0.28 +
        (state.bursting ? extraLen * 0.05 : 0)
      ctx.beginPath()
      ctx.arc(CX, CY, coreR, 0, Math.PI * 2)
      ctx.fillStyle = 'rgba(240,239,232,0.93)'
      ctx.fill()

      state.rafId = requestAnimationFrame(draw)
    }

    stateRef.current.rafId = requestAnimationFrame(draw)

    return () => {
      if (stateRef.current.rafId) {
        cancelAnimationFrame(stateRef.current.rafId)
      }
    }
  }, [])

  // Expose burst trigger
  useEffect(() => {
    if (triggerBurst && isFull) {
      stateRef.current.bursting = true
      stateRef.current.burstT = 0
    }
    // On Tier 2: no burst, just breathing
  }, [triggerBurst])

  return <canvas ref={canvasRef} width={40} height={40} />
}

// The exported component — renders the right version per tier
export default function DandelionIcon({ triggerBurst }) {
  if (isMinimal) {
    return <DandelionStatic />
  }
  return <DandelionCanvas triggerBurst={triggerBurst} />
}
```

The Backdrop-Filter Problem

On Android Chrome it forces a re-composite of everything behind
the blurred element on every frame. On low-end devices it drops
to 15fps or lower.

color system. The result looks different but feels just as premium.

PART NINE — VIRTUAL LIST FOR TASK RENDERING
---
The Problem This Solves
A day with 20+ tasks rendering 20 task cards, all with
will-change: transform, all potentially animating on swipe —
this creates GPU memory pressure on low-end devices.
The fix: only render what is visible.
```jsx
// /src/components/tasks/VirtualTaskList.jsx
// Only render tasks that are in or near the viewport

import { useRef, useState, useEffect, useCallback } from 'react'
import TaskCard from './TaskCard'

const TASK_HEIGHT_ESTIMATE = 56   // px, collapsed card height
const OVERSCAN = 3                 // render 3 extra above and below viewport

export default function VirtualTaskList({ tasks, ...props }) {
  const containerRef = useRef(null)
  const [visibleRange, setVisibleRange] = useState({ start: 0, end: 20 })

  const updateVisibleRange = useCallback(() => {
    if (!containerRef.current) return
    const container = containerRef.current
    const { scrollTop, clientHeight } = container.closest('[data-scroll-container]')
      || document.documentElement

    const containerTop = container.getBoundingClientRect().top + window.scrollY

    const startIndex = Math.max(
      0,
      Math.floor((scrollTop - containerTop) / TASK_HEIGHT_ESTIMATE) - OVERSCAN
    )
    const endIndex = Math.min(
      tasks.length,
      Math.ceil((scrollTop - containerTop + clientHeight) / TASK_HEIGHT_ESTIMATE) + OVERSCAN
    )

    setVisibleRange({ start: startIndex, end: endIndex })
  }, [tasks.length])

  useEffect(() => {
    updateVisibleRange()
    window.addEventListener('scroll', updateVisibleRange, { passive: true })
    return () => window.removeEventListener('scroll', updateVisibleRange)
  }, [updateVisibleRange])

  const totalHeight = tasks.length * TASK_HEIGHT_ESTIMATE
  const visibleTasks = tasks.slice(visibleRange.start, visibleRange.end)
  const topPadding = visibleRange.start * TASK_HEIGHT_ESTIMATE

  return (
    <div ref={containerRef} style={{ height: totalHeight, position: 'relative' }}>
      <div style={{ transform: `translateY(${topPadding}px)` }}>
        {visibleTasks.map((task, i) => (
          <TaskCard
            key={task.id}
            task={task}
            index={visibleRange.start + i}
            {...props}
          />
        ))}
      </div>
    </div>
  )
}
```
Note: only use virtual list when task count exceeds 15.
For typical days (4–8 tasks) render all tasks normally —
virtualization overhead is not worth it for small lists.

PART ELEVEN — THE ANIMATION CONCURRENCY BUDGET
---
Maximum Simultaneous Animated Elements
```
At any given moment in Flow:

Layer 1 (Interaction):  Up to 2 simultaneously
  (e.g., a swipe + the checkmark fill)
  These are always allowed. They are what the user is doing.

Layer 2 (Response):     Up to 4 simultaneously
  (e.g., task creation + day pulse update + badge bump + banner)
  Short-lived. Auto-terminate.

Layer 3 (Ambient):      Up to 4 CSS keyframe animations
  (base color transition + 3 drift layers on Tier 1)
  PAUSED during page transitions and focus mode entry

Layer 4 (Microstate):   Up to 4 simultaneously
  (timeline pulse + sync indicator + focus breathe + dandelion)
  All CSS keyframes on opacity/transform only. Cheapest category.

Total maximum: 14 animated elements at once.
In practice: most of the time it is 4–6.
```
---
The Pause Rules
```
During page transitions (280–400ms):
  → Pause all Layer 3 ambient drift animations
  → Pause Layer 4 microstates (except sync indicator)
  → Resume all after transition completes

During Focus Mode (entire session):
  → Pause all Layer 3 ambient drift animations
  → Pause all Layer 4 microstates except focusBreathe
  → The dandelion dims to 20% opacity (dims, not pauses)
  → Resume all on Focus Mode exit

During drag-to-reorder:
  → Pause Layer 3 ambient drift animations
  → Pause Layer 4 microstates
  → All GPU budget goes to the drag interaction
  → Resume on drop

During device low-power mode (battery saver):
  → Detect via navigator.getBattery() if available
  → If battery < 15% or charging is false + battery < 20%:
    → Treat as Tier 3 regardless of hardware
  → Restart detection on next session (not mid-session)
```

PART TWELVE — PERFORMANCE MONITORING
---
Detecting When to Downgrade Mid-Session
Never change tiers mid-session — it creates visible jarring shifts.
But do monitor frame rate to detect if the current tier is too expensive.
```javascript
// /src/lib/performanceMonitor.js

import { DEVICE_TIER, TIERS } from './deviceTier'

let frameCount = 0
let lastTime = performance.now()
let lowFPSCount = 0
const LOW_FPS_THRESHOLD = 25
const LOW_FPS_STRIKES = 5  // 5 consecutive low readings = downgrade next session

function measureFPS() {
  frameCount++
  const now = performance.now()

  if (now - lastTime >= 1000) {
    const fps = Math.round((frameCount * 1000) / (now - lastTime))
    frameCount = 0
    lastTime = now

    if (fps < LOW_FPS_THRESHOLD) {
      lowFPSCount++
      if (lowFPSCount >= LOW_FPS_STRIKES) {
        // Mark for downgrade on next session launch
        const nextTier = DEVICE_TIER === TIERS.FULL
          ? TIERS.STANDARD
          : TIERS.MINIMAL
        localStorage.setItem('flow_device_tier', nextTier)
        localStorage.setItem('flow_device_tier_time', String(Date.now()))
        // Do NOT apply mid-session — wait for next open
      }
    } else {
      lowFPSCount = 0
    }
  }

  requestAnimationFrame(measureFPS)
}

// Only run monitoring on Standard tier
// Full tier devices don't need monitoring
// Minimal tier already runs cheapest settings
if (DEVICE_TIER === TIERS.STANDARD) {
  requestAnimationFrame(measureFPS)
}
```
